# Lungfish v0.4.0-alpha.5 Release Notes

Previous release: v0.4.0-alpha.4

## Highlights

- Added a ViralRecon mapping workflow with dedicated request models, input validation, primer staging, samplesheet generation, operations-panel logging, and XCUITest fixture coverage.
- Added ribosomal RNA filtering from FASTQ operations and CLI entry points, including provenance capture for reproducible lungfish-cli outputs.
- Normalized FASTQ operation outputs so derived bundles carry workflow provenance and compressed/clumpified FASTQ artifacts where appropriate.
- Hardened async UI state across the app using shared request/selection gates, reducing stale updates in the main window, inspector, database search, viewer fetches, and metagenomics result views.
- Fixed EsViritu detail-view recursion and Kraken2 BLAST drawer races that could crash or hide completed BLAST verification results.
- Fixed NAO-MGS classifier read extraction so accession-scoped extraction counts align with per-accession unique-read accounting.

## Metagenomics And Workflow Changes

- Replaced the generic nf-core workflow surface with a focused ViralRecon workflow path.
- Improved ViralRecon process output draining, cancellation behavior, operation logs, and recovery from invalid wizard selections.
- Added provenance support for lungfish-cli workflow commands, including RiboDetector, FASTQ operations, and generated output bundles.
- Improved workflow command previews by centralizing escaping and making preview strings more consistent with actual execution.
- Added batch classifier table filtering and metadata joins for Kraken2, EsViritu, TaxTriage, and NAO-MGS views.
- Preserved metagenomics action targets and visible result selection identity across filtering, sorting, and UI refreshes.

## Stability Fixes

- Prevented stale database workflow import updates from replacing newer UI state.
- Scoped inspector notifications and main-window routing so unrelated windows or stale selections do not receive updates.
- Invalidated stale sequence-viewer fetches when content or read-display settings change.
- Fixed NVD context actions so right-click actions bind to the clicked contig.
- Hardened tar extraction and process-stream handling for metagenomics workflows.
- Fixed XCUITest runner behavior when running without test filters.

## Release And Maintenance

- Added state-management hardening plans and tests for request gates, validation sessions, selection identity stores, and window-state scoping.
- Updated release notes and version metadata for v0.4.0-alpha.5.
- Updated the in-app Release Notes menu link to the current GitHub repository.
